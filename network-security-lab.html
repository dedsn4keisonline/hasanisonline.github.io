Overview

As a cybersecurity enthusiast, I wanted a safe environment to test firewalls, intrusion detection systems (IDS), and penetration testing tools. That’s why I built a network security lab. It’s a simulated environment where I can experiment without risking real-world systems. Let me walk you through how I did it!

My Network Security Lab

What I Built

Here are the key components of my network security lab:

  • Firewall Testing: I set up pfSense to simulate and test firewall rules.
  • Intrusion Detection: I deployed Snort as an IDS to monitor network traffic.
  • Penetration Testing: I used Kali Linux and tools like Metasploit for ethical hacking.
  • Virtualization: I used VMware ESXi to create isolated virtual networks.

How I Built It

1. Setting Up the Virtual Environment

I started by creating a virtual environment using VMware ESXi. Here’s how I did it:

  1. I installed VMware ESXi on a dedicated server.
  2. I created virtual machines (VMs) for different network components (e.g., firewalls, IDS, and targets).
  3. I configured virtual networks to isolate traffic for testing.

VMware ESXi Dashboard

2. Configuring pfSense Firewall

Next, I set up pfSense as the firewall for my lab. Here’s my process:

  1. I installed pfSense on a VM and configured it as the gateway for my virtual network.
  2. I created firewall rules to simulate different scenarios (e.g., blocking specific traffic).
  3. I tested the rules to ensure they worked as expected.

pfSense Dashboard

3. Deploying Snort IDS

To monitor network traffic, I deployed Snort as an intrusion detection system. Here’s how I did it:

  1. I installed Snort on a VM and configured it to monitor traffic on the virtual network.
  2. I created custom rules to detect specific types of attacks (e.g., port scans, SQL injection).
  3. I tested Snort by simulating attacks and verifying alerts.

Snort Alerts

4. Penetration Testing with Kali Linux

Finally, I used Kali Linux to test the security of my lab. Here’s my process:

  1. I installed Kali Linux on a VM and configured it with tools like Metasploit, Nmap, and Wireshark.
  2. I simulated attacks on the virtual network to test the firewall and IDS.
  3. I analyzed the results and made improvements to the security setup.
